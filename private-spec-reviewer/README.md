# Spec Reviewer — Customer-Side deployment

## Overview

This module provisions an isolated browser session environment inside **your** AWS account so Baz can review your application without ever accessing your credentials.

It deploys:
- An **AgentCore Browser** (optionally inside your VPC to reach private staging environments)
- An **AgentCore Runtime** that pulls credentials from your SSM Parameter Store, logs in to your preview environment, and returns a browser session handle
- **IAM roles** that let Baz invoke the runtime and connect to the browser cross-account, scoped by a generated `external_id`

Your username and password never leave your AWS account. The only thing returned to Baz is `{browser_id, session_id}`.

---

## Prerequisites

- AWS account with Terraform ≥ 1.5
- The `handler_image_uri` provided by Baz
- SSM parameters for your preview environment credentials already created

---

## Quick start — public browser

Suitable for publicly accessible preview environments.

```hcl
module "baz_spec_login" {
  source             = "./customer_spec_login"
  baz_aws_account_id = "<provided by Baz>"
  handler_image_uri  = "<provided by Baz>"
  ssm_username_path  = "/myapp/staging/username"
  ssm_password_path  = "/myapp/staging/password"
}

output "cross_account_role_arn" { value = module.baz_spec_login.cross_account_role_arn }
output "runtime_arn"            { value = module.baz_spec_login.runtime_arn }
output "browser_id"             { value = module.baz_spec_login.browser_id }
output "external_id"            { value = module.baz_spec_login.external_id sensitive = true }
```

---

## Private environments (VPC mode)

When your preview environment is only reachable from inside a VPC, supply `vpc_id`, `subnet_ids`, and `preview_env_cidr`. The browser is provisioned with ENIs in your private subnets and can reach the environment directly — no NAT gateway required.

```hcl
module "baz_spec_login" {
  source             = "./customer_spec_login"
  baz_aws_account_id = "<provided by Baz>"
  handler_image_uri  = "<provided by Baz>"
  ssm_username_path  = "/myapp/staging/username"
  ssm_password_path  = "/myapp/staging/password"

  vpc_id           = "vpc-0abc123"
  subnet_ids       = ["subnet-0aaa", "subnet-0bbb"]
  preview_env_cidr = "10.0.0.0/16"
}
```

## What stays in your account

| Data | Where it lives |
|---|---|
| Username / password | Your SSM Parameter Store — never returned to Baz |
| Session recordings | Your S3 bucket (`baz-browser-recordings-{account_id}`) |
| Credentials in transit | Fetched inside the runtime, inside your account |
| Returned to Baz | `browser_id` + `session_id` only |

Baz connects to the live browser session via a public AWS CDP endpoint using the session handle. Your credentials are never part of that connection.

---

## Outputs

After `terraform apply`, paste these values into Baz's dashboard under **Settings → Spec Login**:

| Output | Where to use |
|---|---|
| `cross_account_role_arn` | "Cross-account role ARN" field — alongside `external_id` |
| `runtime_arn` | "Runtime ARN" field |
| `browser_id` | "Browser ID" field |
| `external_id` | "External ID" field — generated by Terraform, supplied by Baz on every `AssumeRole` call |

## PrivateLink (optional)

Set `enable_privatelink = true` (requires VPC mode) to route all AgentCore API calls — `StartBrowserSession`, `InvokeAgentRuntime`, etc. — through a VPC interface endpoint. Control-plane traffic never traverses the public internet.

```hcl
module "baz_spec_login" {
  # ...VPC mode variables...
  enable_privatelink = true
}
```

No changes to the runtime code or Baz application are required.
